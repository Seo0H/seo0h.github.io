공부했던 책은 **필기는 시나공**, 실기는 **시나공 퀵이지 정보처리기사 실기 단기완성** 이에요. 여러 정처기 책 중 시나공을 선택한 이유는.. 시나공 필기의 경우 작년 교육 마무리하며 꽁으로 받은 책이기 때문이에요.  ~~공짜 최고!.. 앗 혹시 뒤로가시는건 아닌지 걱정되네요. 🫢~~

사실 책의 경우 **어떤 브랜드 책을 사는지는 별로 중요하지 않다고 생각**해요. 개념 자체는 인터넷에 다 풀려있어서 좀만 서치 해보시면 필수개념 PDF 같은 자료 충분히 찾으실 수 있으실 거에요.

그래도 **실기 책의 경우 선택한 이유가 있어요**. 실기를 3주정도 기간을 두고 준비를 했기 때문에 기존 그 두꺼운 실기 책을 다 볼 엄두가 안났었어요. 그래서 **최대한 얇지만 문제가 들어있는 책**을 찾다가 **시나공 퀵이지**를 사게 되었어요. 개념의 경우 문제은행을 돌리던지 검색하면 다 나오지만, **문제를 직접 손으로 풀어보는 건 필요하다는 생각**에 내린 결정이였어요.

## 공부 방법
### 필기

> **정처기 필기 특징** <br />
> - CBT(Computer Based Test) 형식
> - 문제은행
> - 4지선다. 객관식
> - 시험 종료 후 바로 점수 확인 가능

> **정처기 필기 공부 방법** <br />
> - 개괄적 이해
> - 교재 2회독 진행
> - [CBT](https://www.comcbt.com/) 기출 풀이

필기는 [**CBT**](https://it.eduwill.net/it2/Info/ExamInfo/cbt.asp)로 시험이 이루어지고, 문제은행 형식의 **객관식**이에요. 따라서 어떤 개념을 깊게 알기 보다는 마인드맵 처럼 관련 **하나의 개념을 보면 연속적으로 관련 개념이 머릿속에 남도록 정리하는게 중요**하다 생각했어요.

| 2회독 필기                                                                        | 노션 토글 사용                            |
| --------------------------------------------------------------------------------- | ----------------------------------------- |
| ![정말 나만 이해할 수 있는 방법으로 필기..](/posts/certificate/231029-010435.png) | ![](/posts/certificate/231029-010540.png) |


필기는 있는 책을 일단 2회독까지는 하면서 단어나 개념에 익숙해지려고 했어요. 1회독은 한주에 한 장씩 넉넉하게 잡고 쭉 읽었어요.

2회독부터는 확실히 머리에 남긴다는 생각으로 한 챕터 읽으며 **노션에 중요 포인트를 필기**하는 식으로 접근했어요. 각잡고 필기하는게 아니라, **머릿속에 넣는다는 느낌**으로요. 이때 **노션의 토글 기능**을 정말 잘 이용했어요. 개념 단어 토글안에 해당 개념 설명을 적어둔 후, 틈틈히 이동하며 나름 퀴즈 아닌 퀴즈를 풀었어요. (사실 마지막 4,5 과목은 힘이 딸려서 제대로 2회독까지 보진 못했지만,, 그래도 합격 하더라구요?)

막바지 일주일 남기고는 다들 추천하는 **[CBT](https://www.comcbt.com/) 풀이를 진행**했어요. 이동 시간 내내 풀고 풀고 했던 것 같네요. 정처기가 **20년도 부터 경향이 완전 바뀌어서, 20년도 이후 문제로 푸는 걸 추천**드려요

![시험 전 찐 암기 노트](/posts/certificate/231029-011526.png)

시험 전에는 정말 중요하고 나오면 무조건! 맞아야 하는 것과 계속 햇갈리던 것들만 정리한 노션 문서를 만들고 시험장 가면서 계속 봤어요.

요정도 하니, CBT에서 예상했던 점수 그대로 잘 맞고 나왔어요.

정처기에서 무조건 자주 나오는 개념들을 다 맞고자 하면 합격 점수가 딱 나오도록 되어있는 것 같아요. 문제은행이기도 하구요.. 그러니 **아는 부분에서 햇갈리는 부분이 없도록 만드는게 중요**한 것 같아요. (디자인 패턴, UML, 객체지향 방법론, 럼바우, SOLID 등..)

아래는 제가 느끼기에 각 언어가 요구하는 바와 그에 따른 저의 공부 방법에 대해 정리해 둔 글인데요, 시험을 한번만 준비해본.. 저의 개인적인 의견이니 참고만 하시길 바래요!

- **C** :  **기본적인 '프로그래밍' 개념에 대해 아는지?**
    - 배열 처리, 함수로 인자 넘겨서 계산하면 나오는 값이 무엇인지 처럼 **계산 결과로 어떤 값이 나올지 알 수 있어야 해요.**
    - 당시에 시간이 없어서 총 영상 평균길이와 각 배속에 따른 소요 시간을 계산해 둔게 있네요. 해당 재생목록을 다 보려면 대략 13시간정도 필요하고, 1.5배속으로 보면 8시간정도 필요해요.
    - 비트연산(특히 2의 보수...), 포인터 관련된 부분이 어려웠던 기억이 있는데, 해당 부분은 시간을 충분히 가지고 공부하시길 바라요!
- **Java** :  **객체지향 개념을 아는지?**
    - 객체지향 관련 개념을 Java를 통해 주로 물어보더라구요.
    - 저는 정처기 준비 이전에 **생활코딩 님의 Java 강의**를 들었는데요, 해당 강의 덕에 객체지향에 대한 대략적 개념을 잡을 수 있었어요. 정말정말 쌩 노베이스이시면 해당 강의를 통해 기본적인 객체지향 개념에 대해 잡아두시는걸 추천드려요.
    - 이후 정처기를 준비하는 3주동안 **흥달쌤 영상답변 중 Java 관련된 QnA**를 주로 봤었어요.
    - 제가 봤던 23년도 2회차 시험에서는.. Java의 `String`과 `new String` 을 이용해 생성한 변수의 차이에 대해 알아야 풀 수 있는 문제가 나오긴 했는데.... 이 문제는 정말 Java 언어를 자주 사용해보신 분들만 맞추실 수 있는 문제라고 생각해요. 그러니 강의만 듣고 끝나는게 아닌, 직접 **온라인 IDE에서 코드를 많이 쳐보는게 중요**하다고 더욱더 느꼈어요. 궁금한게 있으면 직접 테스트도 해보고요.
- **Python** : **Python의 특이한 문법(?)을 아는지?**
    - 파이썬은 C언어나 Java와는 문법이 조금 다른데요, 제가 느끼기에는 일단 타입이 유동적이라는 점, 특정 자료구조의 처리, 입출력 처리, 반복문 등 이것저것 기존 정적 언어들과는 달리 **특이한 문법(?)이 있다는 점 정도가 출제 포인트라고 느꼈어요.**
    - 이 파이썬도 흥달쌤이 따로 찍어두신 강의는 없어서 정처기 실기나 필기 책에서 기본적으로 나와있는 개념과 더불어 **흥달쌤 영상답변 QnA에 올라온 파이썬 관련된 영상**을 다 봤던 것 같아요. 영상에서 관련 문제도 같이 나오니 보다가 잠깐 멈춰서 손으로 풀고 답 확인하고 하는 식으로 했었어요.
- **SQL** : **DB의 구조에 대해 알고 있는지? 또한 DB에서 SQL을 이용해 원하는 값을 추출해 낼 수 있는지?**
    - 이것도 역시 위의 방법과 같게 기본 개념은 실기 책으로 익히고, 추가 어려운 개념은 흥달쌤 영상답변을 봤었어요.

추가로 프로그래밍 이외의 계산 문제는 [**서브넷 | 서브넷팅**](https://www.youtube.com/watch?v=lfB_xjlhrUo&feature=youtu.be), **프로세스 스케쥴링 방식**, **개발 기간 산정**, **페이지 교체**, **디스크 스케줄링**, **Fan-in/ Fan-out**, **mccabe 복잡도** 등이 있었던 것으로 기억해요. 역시 역시 흥달쌤 유투브에도 일부 내용이 올라와 있고, 구글에도  검색해보시면 양질의 글과 영상을 찾으실 수 있을거에요.
