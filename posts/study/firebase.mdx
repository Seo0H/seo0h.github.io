외주 메뉴얼을 작성하며 계속 머릿속에 맴돌던 문제가 있었다. 그건 바로 관리자 페이지였다. 나에게 의뢰한 개발을 의뢰한 사람들이 요구할 확률이 많다고 생각이 들었고, 이번 기회에 back-end 구축 없이 기존에 있는 기술들을 이용해 최대한 관리자 페이지를 만들어 봐야 겠다고 생각했다.

내가 들어가기 전에 생각했던 관리자 페이지의 기능이다.
```
1. 팝업 이미지를 커스텀 할 수 있으며, 팝업의 기한도 설정 할 수 있음.
2. 관리자 글 쓰기 기능이 있어서 글을 쓰고 그걸 웹에 게시할 수 있음.
```

따라서, 관리자임을 알 수 있는 로그인 기능이 필요함을 느꼈다. 로그인과 관리자 인증 부분은 요즘 잘 되어 있는 소셜 로그인 기능을 이용해 구현할 수 있을 것이고, 게시판 기능은 여러 옵션이 있을 것 같았다.

내가 따로 back-end를 만들 수 있는 상황이 아니고 만들어도 여러 문제가 있을 것이 뻔하기에, 여러 기업에서 제공하는 DB 서비스를 이용하기로 했다. 대표적으로 `firebase`가 있었고 가격 옵션도 넉넉하게 무료이기에 해당 기술을 사용하기로 결정했다.
