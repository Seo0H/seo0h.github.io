MDX?

이 블로그 글은 MDX로 작성되었어요. 마크다운(.md)으로 글을 작성하면, 가독성 있는 글을 빠르게 작성할 수 있기 때문에 많은 사람들이 선호하는 도구에요.

그 중 MDX는 MD + JSX를 합친 단어로, JSX를 마크다운에서 작성할 수 있게 해줘요. 따라서 MDX를 이용해 글을 쓰면, 마크다운 형식으로 글을 작성하며 React 컴포넌트를 포함시킬 수 있어요.

MDX와 Next.js

현재(23.10.12) Next js의 공식문서의 MDX 챕터에서는

  1. 로컬 데이터를 이용하는 경우 : @next/mdx
  2. 원격 데이터를 사용하는 경우 : next-mdx-remote, contentlayer

이렇게 두가지에 대해 대안을 제안하고 있어요.

여기서 의미하는 로컬이란, /pages 또는 /app 하위에 생성되는 페이지들이 .mdx로 작성되는 것을 의미하며, /pages, /app 바깥쪽에 mdx파일이 위치하는 경우 원격이라고 정의해요.

  your-project
  ├── pages
  │   └── my-mdx-page.mdx
  └── package.json
로컬인 상황 파일 구조 예시

제가 원하는 블로그의 완성 조건은 다음과 같았어요.

  1. 첫 화면에 소개와 함께, 블로그 포스트들이 뜨도록 하는 것
  2. 각 포스트 클릭 시 동적인 라우팅을 통해 해당 페이지에 맞는 데이터를 가진 데이터 매칭 구현
  3. 공통적인 레이아웃 공유 (헤더와 푸터, 각 포스트 등..)
  4. 이후의 다른 프로젝트와 확장 가능성 염두

각 페이지가 마크다운 그 자체가 되는 것이 좋지만 4번, 확장 가능성 때문에 포스트의 관심사를 다른 곳에 분리하고 싶었고, 따라서 contentlayer를 사용하게 되었어요.

next-mdx-remote VS contentlayer

왜 next-mdx-remote 가 아닌 contentlayer를 사용하게 된 이유는 아래와 같아요.

  1. 데이터 스키마 명시적 작성 강제
  2. 위 스키마에 따라 빌드 시 TypeScript 유형 정의 생성

이 두가지를 큰 장점으로 뽑은 이유는 블로그 글을 작성하면 일정한 탬플릿이 필요하기 때문이에요. 해당 글의 제목, 부제목 이외에도 글에 대한 메타 정보들이 포함될 수 있는 일정한 양식 말이죠. 이런 메타 정보는 아래와 같은 형식으로 작성되요. --- 사이에 key - value 형식으로 작성한답니다.

---
title:'my-title'
description:'my-description'
...
---
마크다운에서 에서 메타데이터 표기 방식

이런 정보들을 마크다운을 작성할때마다 복사 붙여넣기 하며 작성하는건 꽤 번거로운 일 일거예요. 물론 자동완성 기능을 제공하는 code snippets도 사용하겠지만, Contentlayer 에서 정보의 스키마를 정의해두고 검사해준다면, 해당 정보를 잘못 작성하거나 빼먹을 확률이 적어지겠죠?

친절하게 문제를 알려주는 contentlayer..

🔖 주의사항
Contentlayer는 아직 beta버전이에요. 이 글 작성 기준(2023-10-12), 지원되는 프레임워크는 Next.js 뿐이며 지원되는 데이터 소스도 MDX 뿐이에요. 현재 실험적으로 노션으로 컨텐츠를 가져올 수 있어요.

설치 및 초기 환경 세팅

이제 본격적으로 설치 및 초기 환경 세팅을 해볼까요? Contentlayer의 공식 문서에도 자세히 나와있지만, 빠르게 설치와 초기 세팅을 하는 법이 궁금하시다면, 아래 토글을 눌러주세요.

✈️ Contentlayer 빠르게 시작하기

1. 의존성 설치하기

npm install contentlayer next-contentlayer date-fns

2. next config를 contentlayer로 래핑하기

// next.config.js
const { withContentlayer } = require('next-contentlayer')
 
/** @type {import('next').NextConfig} */
const nextConfig = { reactStrictMode: true, swcMinify: true }
 
module.exports = withContentlayer(nextConfig)

3. tsconfig 에 contentlayer 설정 추가하기

// tsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",
    //  ^^^^^^^^^^^
    "paths": {
      "contentlayer/generated": ["./.contentlayer/generated"]
      // ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
    }
  },
  "include": [
    "next-env.d.ts",
    "**/*.ts",
    "**/*.tsx",
    ".next/types/**/*.ts",
    ".contentlayer/generated"
    // ^^^^^^^^^^^^^^^^^^^^^^
  ]
}

4. gitignore에 contentlayer 추가하기

# .gitignore
 
# ...
 
# contentlayer
.contentlayer

데이터 스키마 만들기

기본 설정이 끝났다면, Contentlayer에서 제공하는 defineDocumentType API를 이용해 컨텐츠의 데이터 스키마를 만들어줍니다. 저의 경우, 포스트의 상단의 데이터 필드를 아래와 같은 식으로 짜주었어. 제가 사용한 필드 이외의 속성들이 궁금하다면, 공식문서인 Field Types reference 해당 링크를 방문해보세요.

const fields: FieldDefs = {
  title: { type: 'string', required: true },
  description: { type: 'string', required: true }, // 한줄 요약
  date: { type: 'date', required: true },
  draft: { type: 'boolean' }, // 초고 판별 속성
  image: { type: 'string', default: '/img/default-thumbnail.png' }, // 썸네일
};
field type 정의

위 필드의 양식대로 mdx 포스트를 작성한다면 아래와 같은 mdx 헤더를 사용할 수 있어요.

---
title: 'Next.js MDX 파싱하기'
description: 'Contentlayer을 이용한 MDX 블로그 환경 구축기'
draft: false
image: '/img/2023/blog-making/thumbnail.png'
date: 2023-10-12 19:32:17
---
fields 표기 예시

이렇게 만든 필드를 defineDocumentType 메소드에 인자로 넣어주어야 해요.

const Post = defineDocumentType(() => ({
  name: 'Post', // 필수. 해당 이름으로 문서의 type이 생성됨
  filePathPattern: `**/*.mdx`,
  contentType: 'mdx',
  fields, // 위에서 만든 fildes
  computedFields: {
    url: { type: 'string', resolve: (post) => `/${post._raw.flattenedPath}` },
    readingTime: {
      type: 'string',
      resolve: (post) => Math.ceil(readingTime(post.body.raw).minutes),
    },
    formattedDate: { type: 'string', resolve: (post) => dayjs(post.date).format('YY.MM.DD') },
  },
}));

computedFields 를 이용해 콘텐츠 소스를 인자로 받아 계산해서 추가 필드를 만들 수도 있어요. 저의 경우 현재 url, readiingTime, formattedDate 를 만들어 사용중이에요.(readingTime과 dayjs 패키지를 이용했어요.)

화면에 표시되는 메타 정보들이 추가로 필요한 경우, 이 기능을 이용하면 좋겠죠?

// contentlayer.config.ts
export default makeSource({
  contentDirPath: 'posts', // 필수. 콘텐츠 위치 경로
  documentTypes: [Post], // 필수. 스키마 정의
  mdx: {
    // ... 추가 플러그인
  },
});

최종적으로 contentlayer.config.ts 에서 makeSource를 이용해 위에서 만든 스키마 객체를 넣어 default export 해주면 돼요.

이제 빌드 시 자동으로 mdx 파일을 찾아 contentlayer가 파싱해준답니다. 동시에 .contentlayer 폴더가 생기는데 해당 폴더 안에 빌드 시마다 업데이트 된 정적 파일과 설정해 준 타입이 생성되게 돼요.

➕ Mdx VS Code 세팅하기

위의 글을 읽으며, 마크다운을 이용해 vs 코드 에디터로 글을 작성하는게 불편하다는 생각이 드실 수 있어요. 하지만, 코드 에디터인 만큼 개발자가 설정할 할 수 있는 요소들이 넘쳐난답니다. 제가 사용중인 mdx 관련 설정을 알려드릴게요.

1. 코드 스니펫 설정하기

상단 메타 정보를 매번 작성하는게 너무 귀찮다는 생각이 들 수도 있어요. 왜냐면 저도 그랬거든요! 역시 bepyan님 블로그에서 참고한 것을 제 입맛대로 좀 바꾸어서 사용해봤어요.

// .vscode
{
"Post Matter": {
    "prefix": ["---"],
    "body": [
      "---",
      "title: '$1'",
      "description: '$2'",
      "draft: false",
      "image: '$3",
      "date: $CURRENT_YEAR-$CURRENT_MONTH-$CURRENT_DATE $CURRENT_HOUR:$CURRENT_MINUTE:$CURRENT_SECOND",
      "---",
      "",
      "$0"
    ],
    "scope": "md,mdx"
  }
}

해당 코드 스니팻 문법은 VS Code 공식문서 : Snippets in Visual Studio Code 이 문서에 자세히 나와 있어요.

2. 키 바인딩 설정하기

추가로 마크다운과 같이 사용해보니 편했던 키 바인딩도 알려드릴게요. 키 바인딩은 각 프로젝트(폴더) 별로 오버라이딩 해서 사용하는 것은 불가능하기 때문에, 전역적으로 설정해서 사용해야 해요. 하지만, 조건절을 이용해 특정 파일인 경우만 해당 키 바인딩을 활성화 할 수 있어요.

참고로 키 바인딩은 vscode에서 f1을 누른 후 기본 설정 : 바로 가기 키 열기 (JSON) 을 누르면 설정 가능한 Json 문서가 열려요.

  1. 마크다운 캡션 링크 문법 키바인딩
    [caption](links) 와 같은 캡션-링크 문법을 쉽게 만들어줘요.
// keybindings.json
[
  //...
   {
    "key": "ctrl+v",
    "description": "Smart link paste that works in Markdown",
    "command": "runCommands",
    "args": {
      "commands": [
        {
          "command": "editor.action.insertSnippet",
          "args": {
            "snippet": "[$TM_SELECTED_TEXT]($CLIPBOARD)"
          }
        }
      ]
    },
    "when": "resourceFilename =~ /(.md)/ && editorHasSelection" // 파일이 마크다운 이면서, selection이 참인경우만 활성화
  },
]
마크다운 링크 붙여넣기 문법 구현 키바인딩
  1. 마크다운 캡션 볼드체 적용 키바인딩 
// keybindings.json
[
  //...
   {
    "key": "ctrl+b",
    "description": "bold text maker in markdown",
    "command": "runCommands",
    "args": {
      "commands": [
        {
          "command": "editor.action.insertSnippet",
          "args": {
            "snippet": "**$TM_SELECTED_TEXT**"
          }
        }
      ]
    },
    "when": "resourceFilename =~ /(.md)/ && editorHasSelection"
  }
]

VS코드 키 바인딩 문법은 VS Code 공식 문서 : Key Bindings for Visual Studio Code에 들어가보면 더 자세히 나와있답니다.
